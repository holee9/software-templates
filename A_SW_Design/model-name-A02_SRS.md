### 4.2 SW 제품 개요

model-name Console SW는 DR(Digital Radiography) 검출기와 연결된 의료 영상 획득·표시·후처리 소프트웨어이다.  
주요 기능은 DICOM 영상 획득, 표시, 처리(밝기/대비/변환), 주석(측정), PACS 전송이며, Windows 10/11 64-bit 환경에서 동작한다.

**의도된 사용 목적**: 의료 기관 내 방사선 기사 및 방사선과 전문의가 DR 검출기로 획득한 X-ray 영상을 검토, 처리하고 PACS로 전송하는 데 사용한다.

## 5. 비고: 최소 필수 vs 리스크 최소화 선택 가이드

**왜 최소 필수인가?**  
SRS는 "무엇을 만들었는가"의 증거다. FDA 510(k)에서 SW 기능 설명은 실질적 동등성(Substantial Equivalence) 판단의 핵심이며, MFDS는 기능 목록 제출을 의무화한다. EU MDR Annex I §17.1도 의도된 목적에 맞는 SW 설계 명세를 요구한다.

**없으면 어떤 리스크?**  
- **FDA**: SW 기능 미명시 → eSTAR 심사 불가, Additional Information 요청 확실  
- **MFDS**: 허가 신청 시 기능 목록 반드시 제출 — 미작성 시 반려  
- **EU MDR**: Notified Body 기술 문서 §3 (Design and Manufacturing Information) 불충족  

**작성 팁**:  
- 요구사항 ID를 처음부터 일관되게 부여하면 RTM(model-name-B03) 작성이 간단해진다.  
- 스프레드시트(Excel/Google Sheets)와 병행 관리하면 추적성 매트릭스 생성이 용이하다.  
- 안전 요구사항(SWR-SAF)은 반드시 위험 관리 파일(model-name-D01)과 동기화해야 한다.
